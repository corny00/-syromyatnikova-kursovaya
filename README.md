
Десктопное приложение на WPF для автоматизации работы штрафстоянки. 

Система позволяет оформлять приём эвакуированных транспортных средств, вести учёт хранения, рассчитывать стоимость услуг, формировать отчётность и генерировать официальные документы.

---Основные возможности:

1.Авторизация пользователей с разграничением прав доступа (Администратор, Диспетчер, Инспектор)
2.Электронный реестр задержанных транспортных средств с фильтрацией и поиском
3.Оформление приёма автомобиля: внесение данных ТС, фиксация повреждений, чек-лист комплектации, загрузка фотографий
4.Карточка автомобиля с фотогалереей, таймером времени хранения и автоматическим расчётом стоимости
5.Процедура выдачи ТС: проверка документов, выбор способа оплаты, формирование акта приёма-передачи в формате DOCX
6.Аналитические отчёты: финансовые показатели, статистика по инспекторам, детализация операций с экспортом в CSV
7.Администрирование: управление пользователями, настройка тарифов по категориям ТС
8.Генерация официального акта приёма-передачи с подписями и печатью

---Роли пользователей
Администратор: полный доступ ко всем функциям системы, включая управление пользователями и тарифами.
Диспетчер: работа с реестром, приём и выдача автомобилей, формирование отчётов. Не имеет доступа к администрированию.
Инспектор ГИБДД: только просмотр реестра и данных о направленных на стоянку автомобилях. Не может принимать, выдавать или редактировать записи.

---Технологии
Язык: C#
Платформа: WPF (.NET Framework / .NET 6+)
База данных: Microsoft SQL Server
Доступ к БД: ADO.NET (SqlConnection, SqlCommand, SqlDataAdapter)
Генерация документов: Xceed DocX (библиотека для работы с DOCX)
Архитектура: слоистая, с выделением репозиториев для работы с данными


---Структура базы данных
Основные таблицы:
EvacuationRegistry — реестр задержанных транспортных средств
VehicleOwners — владельцы транспортных средств
CarMakes, CarModels, CarColors, VehicleCategories — справочники ТС
TowTrucks — эвакуаторы
Users, Roles — пользователи и роли
Tariffs — тарифы на услуги
Damages, DamageZones, DamageTypes — повреждения и их классификация
ImpoundChecklist, ChecklistDefinitions — чек-лист комплектации
VehiclePhotos — фотографии автомобилей
Payments, PaymentMethods — оплата и способы оплаты
ReleaseVerifications — проверка документов при выдаче


---Установка и запуск
Разверните базу данных: выполните SQL-скрипт создания схемы в SQL Server
Откройте решение в Visual Studio
Проверьте строку подключения в App.config (ключ GibddConnection)
Установите пакет через NuGet: Install-Package DocX
При необходимости активируйте лицензию Xceed в App.xaml.cs
Запустите проект
Настройка подключения к БД
В файле App.config укажите актуальные параметры:
<connectionStrings>
<add name="GibddConnection"
connectionString="Server=.\SQLEXPRESS;Database=GBDD_BD;Trusted_Connection=True;"
providerName="System.Data.SqlClient"/>
</connectionStrings>

---Формирование документов
Акт приёма-передачи генерируется в формате DOCX с использованием библиотеки Xceed. Документ содержит:
Шапку с реквизитами утверждающего органа
Данные транспортного средства и владельца
Список зафиксированных повреждений
Номер и дату протокола задержания
Отметку о претензиях к хранению
Дату, время и подписи сторон
Файл автоматически сохраняется на рабочий стол и открывается в программе по умолчанию.

---Экспорт данных
Отчёты можно экспортировать в формате CSV для дальнейшей обработки в Excel. Файл сохраняется в кодировке UTF-8 с разделителем точка с запятой.
